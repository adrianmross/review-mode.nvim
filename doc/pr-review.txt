==============================================================================
OVERVIEW                                                  *pr-review-overview*

pr-review.nvim keeps GitHub PR review in ordinary file buffers. It compares the
current branch to the PR base, sets Gitsigns to that base when available, loads
review comments asynchronously, and provides commands for jumping through PR
hunks and changed files.

The plugin uses `gh` for PR metadata, comments, and replies.

==============================================================================
COMMANDS                                                  *pr-review-commands*

                                                        *:PrReviewStart*
:PrReviewStart
        Start normal-buffer PR review mode.

                                                        *:PrReviewStop*
:PrReviewStop
        Stop PR review mode and clear plugin state.

                                                        *:PrReviewRefresh*
:PrReviewRefresh
        Reload changed files and review comments.

                                                        *:PrReviewNextChange*
:PrReviewNextChange
        Jump to the next PR hunk, wrapping across files.

                                                        *:PrReviewPrevChange*
:PrReviewPrevChange
        Jump to the previous PR hunk, wrapping across files.

                                                        *:PrReviewNextFile*
:PrReviewNextFile
        Jump to the next changed file.

                                                        *:PrReviewPrevFile*
:PrReviewPrevFile
        Jump to the previous changed file.

                                                        *:PrReviewOldToggle*
:PrReviewOldToggle
        Toggle a side-by-side diff split with the base version of the current
        file.

                                                        *:PrReviewThread*
:PrReviewThread
        Show PR comments on the current line.

                                                        *:PrReviewReply*
:PrReviewReply
        Reply to the latest PR comment on the current line.

                                                        *:PrReviewComment*
:PrReviewComment
        Create a PR comment on the current line or visual range.

==============================================================================
SETUP                                                        *pr-review-setup*

>
  require("pr_review").setup({
    auto_open_first_change = true,
    comments = {
      enabled = true,
      cache_ttl_seconds = 300,
    },
    diff = {
      fast_diffopt = "internal,filler,closeoff,indent-heuristic,linematch:0",
      use_fast_diffopt = true,
    },
    gitsigns = {
      enabled = true,
    },
    nvim_tree = {
      enabled = true,
    },
    performance = {
      ui_refresh_debounce_ms = 50,
    },
    commands = true,
  })
<

PrReviewStart loads PR metadata and changed-file status asynchronously. Hunk
locations are loaded lazily per file, so opening review mode does not parse the
entire PR patch up front.

PrReviewOldToggle uses the built-in Neovim side-by-side diff backend. When
diff.use_fast_diffopt is enabled, it temporarily applies diff.fast_diffopt while
creating the base split and restores the previous diffopt when the split closes.
