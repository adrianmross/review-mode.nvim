==============================================================================
OVERVIEW                                                  *pr-review-overview*

pr-review.nvim keeps GitHub PR review in ordinary file buffers. It compares the
current branch to the PR base, sets Gitsigns to that base when available, loads
review comments asynchronously, and provides commands for jumping through PR
hunks, PR comments, and changed files.

The plugin uses `gh` for PR metadata, comments, and replies.

==============================================================================
COMMANDS                                                  *pr-review-commands*

                                                        *:PrReviewStart*
:PrReviewStart
        Start normal-buffer PR review mode.

                                                        *:PrReviewStop*
:PrReviewStop
        Stop PR review mode and clear plugin state.

                                                        *:PrReviewRefresh*
:PrReviewRefresh
        Reload changed files and review comments.

                                                        *:PrReviewNextHunk*
:PrReviewNextHunk
        Jump to the next PR hunk, wrapping across files.

                                                        *:PrReviewPrevHunk*
:PrReviewPrevHunk
        Jump to the previous PR hunk, wrapping across files.

                                                        *:PrReviewNextComment*
:PrReviewNextComment
        Jump to the next PR comment.

                                                        *:PrReviewPrevComment*
:PrReviewPrevComment
        Jump to the previous PR comment.

                                                        *:PrReviewNextFile*
:PrReviewNextFile
        Jump to the next changed file.

                                                        *:PrReviewPrevFile*
:PrReviewPrevFile
        Jump to the previous changed file.

                                                        *:PrReviewOldToggle*
:PrReviewOldToggle
        Toggle a side-by-side diff split with the base version of the current
        file.

                                                        *:PrReviewThread*
:PrReviewThread
        Show PR comments on the current line.

                                                        *:PrReviewReply*
:PrReviewReply
        Reply to the latest PR comment on the current line.

                                                        *:PrReviewComment*
:PrReviewComment
        Create a PR comment on the current line or visual range.

                                                        *:PrReviewViewedToggle*
:PrReviewViewedToggle
        Toggle viewed state for the current PR file.

                                                        *:PrReviewViewedNext*
:PrReviewViewedNext
        Mark the current PR file viewed and jump to the next unviewed file.

                                                        *:PrReviewViewedFeatureToggle*
:PrReviewViewedFeatureToggle
        Toggle viewed-state tracking on or off.

                                                        *:PrReviewCommentsToggle*
:PrReviewCommentsToggle
        Toggle PR comments on or off.

                                                        *:PrReviewViewedList*
:PrReviewViewedList [all|viewed|unviewed]
        Open a quickfix list of PR files by viewed state.

                                                        *:PrReviewViewedClear*
:PrReviewViewedClear
        Clear local viewed state for the current PR.

                                                        *:PrReviewViewedSync*
:PrReviewViewedSync
        Pull viewed state from GitHub for the current PR.

                                                        *:PrReviewViewedSyncToggle*
:PrReviewViewedSyncToggle
        Toggle GitHub viewed-state sync.

                                                        *:PrReviewSummary*
:PrReviewSummary
        Show file, comment, thread, and viewed-sync counts.

PrReviewProcessedToggle, PrReviewProcessedList, PrReviewProcessedClear,
PrReviewProcessedSync, PrReviewProcessedSyncToggle, and
PrReviewProcessingToggle remain available as v0.3.0 compatibility aliases for
the viewed commands above.

==============================================================================
KEYMAPS                                                    *pr-review-keymaps*

Suggested navigation uses hunk keys for changed regions and comment keys for
review discussion:
>
  ]h    next PR hunk
  [h    previous PR hunk
  ]c    next PR comment
  [c    previous PR comment
  ]f    next changed file
  [f    previous changed file
<

==============================================================================
SETUP                                                        *pr-review-setup*

>
  require("pr_review").setup({
    auto_open_first_change = true,
    comments = {
      enabled = true,
      cache_ttl_seconds = 300,
      sign_text = "◆",
      sign_hl_group = "DiagnosticInfo",
      virtual_text = true,
    },
    diff = {
      fast_diffopt = "internal,filler,closeoff,indent-heuristic,linematch:0",
      use_fast_diffopt = true,
    },
    gitsigns = {
      enabled = true,
    },
    nvim_tree = {
      enabled = true,
      show_comments = true,
      show_viewed = true,
    },
    viewed = {
      enabled = true,
      sync = false,
      state_path = nil,
    },
    performance = {
      ui_refresh_debounce_ms = 50,
      hunk_prefetch = {
        enabled = true,
        count = 8,
        concurrency = 2,
        focused_delay_ms = 0,
        gitsigns_delay_ms = 5,
      },
      background_hunk_scan = {
        enabled = true,
        max_files = 5000,
        delay_ms = 250,
      },
    },
    commands = true,
  })
<

PrReviewStart loads PR metadata and changed-file status asynchronously. If
GH_REVIEW_BASE is set by a launcher, changed-file loading starts immediately
without waiting for GitHub metadata. Hunk locations are loaded lazily per file,
with immediate focused-file prefetch, opportunistic gitsigns.nvim hunk-cache
reuse, and an optional delayed background scan for PRs under
performance.background_hunk_scan.max_files.

External launchers can provide GH_REVIEW_REPO, GH_REVIEW_PR, GH_REVIEW_BASE,
and GH_REVIEW_HEAD to avoid startup discovery calls.

Viewed state is persisted in stdpath("state")/pr-review-state.json by default.
Set viewed.sync = true or run :PrReviewViewedSyncToggle to pull GitHub's PR file
viewed state at startup and push local viewed/unviewed toggles back to GitHub.
The v0.3.0 processing config key and nvim_tree.show_processing option remain
accepted as compatibility aliases.

PrReviewOldToggle uses the built-in Neovim side-by-side diff backend. When
diff.use_fast_diffopt is enabled, it temporarily applies diff.fast_diffopt while
creating the base split and restores the previous diffopt when the split closes.
