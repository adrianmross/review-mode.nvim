==============================================================================
OVERVIEW                                                  *review-mode-overview*

review-mode.nvim keeps GitHub pull request review in ordinary file buffers. It compares the
current branch to the PR base, sets Gitsigns to that base when available, loads
review comments asynchronously, and provides commands for jumping through PR
hunks, PR comments, and changed files.

The plugin uses `gh` for PR metadata, comments, and replies.

==============================================================================
COMMANDS                                                  *review-mode-commands*

                                                        *:ReviewMode*
:ReviewMode
        Start normal-buffer Review Mode.

                                                        *:ReviewModeStop*
:ReviewModeStop
        Stop Review Mode and clear plugin state.

                                                        *:ReviewModeRefresh*
:ReviewModeRefresh
        Reload changed files and review comments.

                                                        *:ReviewModeNextHunk*
:ReviewModeNextHunk
        Jump to the next PR hunk, wrapping across files.

                                                        *:ReviewModePrevHunk*
:ReviewModePrevHunk
        Jump to the previous PR hunk, wrapping across files.

                                                        *:ReviewModeNextComment*
:ReviewModeNextComment
        Jump to the next PR comment.

                                                        *:ReviewModePrevComment*
:ReviewModePrevComment
        Jump to the previous PR comment.

                                                        *:ReviewModeNextFile*
:ReviewModeNextFile
        Jump to the next changed file.

                                                        *:ReviewModePrevFile*
:ReviewModePrevFile
        Jump to the previous changed file.

                                                        *:ReviewModeOldToggle*
:ReviewModeOldToggle
        Toggle the base version or unified diff for the current file.

                                                *:ReviewModeDiffLayoutToggle*
:ReviewModeDiffLayoutToggle
        Toggle the open diff between side-by-side and unified layout.

                                                  *:ReviewModeDiffFullToggle*
:ReviewModeDiffFullToggle
        Toggle the open diff between condensed context and full-file context.

                                                        *:ReviewModeThread*
:ReviewModeThread
        Show PR comments on the current line.

                                                        *:ReviewModeReply*
:ReviewModeReply
        Reply to the latest PR comment on the current line.

                                                        *:ReviewModeComment*
:ReviewModeComment
        Create a PR comment on the current line or visual range.

                                                        *:ReviewModeViewedToggle*
:ReviewModeViewedToggle
        Toggle viewed state for the current PR file.

                                                        *:ReviewModeViewedNext*
:ReviewModeViewedNext
        Mark the current PR file viewed and jump to the next unviewed file.

                                                        *:ReviewModeViewedFeatureToggle*
:ReviewModeViewedFeatureToggle
        Toggle viewed-state tracking on or off.

                                                        *:ReviewModeCommentsToggle*
:ReviewModeCommentsToggle
        Toggle PR comments on or off.

                                                        *:ReviewModeViewedList*
:ReviewModeViewedList [all|viewed|unviewed]
        Open a fuzzy PR file menu by viewed state, with additions/deletions and
        a diff preview. Press Space or t to toggle viewed state for the
        selected file.

                                                        *:ReviewModeViewedClear*
:ReviewModeViewedClear
        Clear local viewed state for the current PR.

                                                        *:ReviewModeViewedSync*
:ReviewModeViewedSync
        Pull viewed state from GitHub for the current PR.

                                                        *:ReviewModeViewedSyncToggle*
:ReviewModeViewedSyncToggle
        Toggle GitHub viewed-state sync.

                                                        *:ReviewModeSummary*
:ReviewModeSummary
        Show file, comment, thread, and viewed-sync counts.

==============================================================================
KEYMAPS                                                    *review-mode-keymaps*

Suggested navigation uses hunk keys for changed regions and comment keys for
review discussion:
>
  ]h    next PR hunk
  [h    previous PR hunk
  ]c    next PR comment
  [c    previous PR comment
  ]f    next changed file
  [f    previous changed file
<

==============================================================================
SETUP                                                        *review-mode-setup*

>
  require("review_mode").setup({
    auto_open_first_change = true,
    comments = {
      enabled = true,
      cache_ttl_seconds = 300,
      sign_text = "",
      sign_hl_group = "DiagnosticInfo",
      virtual_text = true,
    },
    diff = {
      fast_diffopt = "internal,filler,closeoff,indent-heuristic,linematch:0",
      full_file = false,
      layout = "side_by_side",
      partial_line_highlights = true,
      unified_context = 3,
      use_fast_diffopt = true,
    },
    gitsigns = {
      enabled = true,
    },
    nvim_tree = {
      enabled = true,
      show_comments = true,
      show_viewed = true,
    },
    viewed = {
      enabled = true,
      sync = false,
      state_path = nil,
    },
    performance = {
      ui_refresh_debounce_ms = 50,
      hunk_prefetch = {
        enabled = true,
        count = 8,
        concurrency = 2,
        focused_delay_ms = 0,
        gitsigns_delay_ms = 5,
      },
      background_hunk_scan = {
        enabled = true,
        max_files = 5000,
        delay_ms = 250,
      },
    },
    commands = true,
  })
<

ReviewMode loads PR metadata and changed-file status asynchronously. If
GH_REVIEW_BASE is set by a launcher, changed-file loading starts immediately
without waiting for GitHub metadata. Hunk locations are loaded lazily per file,
with immediate focused-file prefetch, opportunistic gitsigns.nvim hunk-cache
reuse, and an optional delayed background scan for PRs under
performance.background_hunk_scan.max_files.

The nvim-tree integration marks unviewed changed files and closed parent
folders with `☐ N`, where `N` is the number of unviewed changed files under
that node. Viewed files and closed folders use `✓`, and files and closed
folders with unresolved comments also use ` N`. An open folder hides these
folder markers because its children show the same state inline. A folder
switches to viewed after every changed file under it is viewed.

External launchers can provide GH_REVIEW_REPO, GH_REVIEW_PR, GH_REVIEW_BASE,
and GH_REVIEW_HEAD to avoid startup discovery calls.

Viewed state is persisted in stdpath("state")/review-mode-state.json by default.
Set viewed.sync = true or run :ReviewModeViewedSyncToggle to pull GitHub's PR file
viewed state at startup and push local viewed/unviewed toggles back to GitHub.

ReviewModeOldToggle uses the built-in Neovim side-by-side diff backend by
default. Set diff.layout = "unified" or run :ReviewModeDiffLayoutToggle to use an
inline unified diff buffer in the current window instead. Closing unified mode
restores the original file buffer. Set diff.full_file = true or run
:ReviewModeDiffFullToggle to show full-file context. Condensed unified diffs use
diff.unified_context common lines around each hunk, and condensed side-by-side
diffs fold unchanged regions in both diff windows. When diff.use_fast_diffopt is
enabled, side-by-side diffs temporarily apply diff.fast_diffopt and restore the
previous diffopt when the split closes. Unified diffs highlight changed spans
inside modified lines with DiffText; set diff.partial_line_highlights = false
to disable those inline spans.
